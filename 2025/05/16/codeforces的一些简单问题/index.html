Codeforces上的一些简单问题

Posted on 2025-05-16 Edited on 2025-05-26

Abstract:一些有意思的Codeforces 有意思的简单小问题

Apples in Boxes

题目链接：text url
题意：给定长度为$ n $的数列$a$，Tom和Jerry轮流进行游戏，Tom先手，每次选择一个$a_i>0$，使得 $a_i$ 减小 $1$。
若没有$a_i>0$则当前操作者输，若操作后数列极差小于$k$，操作者输。计算谁会赢。
两人操作均为理智的。
思路：为了避免极差大于$k$使得极差尽可能小，最优选择是将数列最大项减$1$，极差单调递减。
设极差为$R$，若一开始$R-1>k$，即Tom操作一次就输。
或是若$R>k$且有多个相等的最大项，则Tom必输。
或是若无论如何$k>=R$，那么当数列之和为偶数时，Tom必输。

Lady Bug

题目链接：https://codeforces.com/problemset/problem/2092/B
题意：给定两个01串，有操作：交换当前字符与斜对角的字符，求能否是第一个变为全0的字符串
思路：分两组，统计第一行的奇数项和第二行的偶数项，第一行的偶数项和第二行的奇数项，1的数量不超一半即可
准确来讲第一组0的数量必须大于$\geq\frac{n+1}{2}$,第二组0的数量则$\geq\frac{n}{2}$

Large Array and Segments

题目链接：https://codeforces.com/problemset/problem/2086/B
给定一个由 $n$ 个正整数组成的数组 $a$，以及一个正整数 $k$。根据以下规则从数组 $a$ 创建数组 $b$：

  • 数组 $b$ 包含 $n \cdot k$ 个元素；
  • 数组 $b$ 的前 $n$ 个元素与数组 $a$ 相同，即对于 $i \le n$，有 $b_{i} = a_{i}$；
  • 对于任意 $i > n$，有 $b_{i} = b_{i - n}$。

例如，若 $a = [2, 3, 1, 4]$ 且 $k = 3$，则 $b = [2, 3, 1, 4, 2, 3, 1, 4, 2, 3, 1, 4]$。

给定一个数 $x$，要求统计满足以下条件的位置 $l$（$1 \le l \le n \cdot k$）的数量：存在位置 $r \ge l$，使得数组 $b$ 在区间 $[l, r]$ 上的元素之和不小于 $x$（即 $b_{l} + b_{l+1} + \dots + b_{r} \ge x$）。
思路：贪心的，对于任意左区间端点$l$,当右端点$r=nk$时，$\sum_{i=l}^{r}B_{i}$最大(显然，不证明)
所以符合条件的最大$l$就是存在的数量
由于$l$越大总和越小，$\sum_{i=l}^{r}B_{i}$不能小于$x$,因此左右端点之和不能小于$(\sum_{i=l}^{r}B_{i})-x$,除法计算有几个完整的数组$a$，以及确定零散的位置。

Permutation Warm-Up

题目链接：https://codeforces.com/contest/2108/problem/A
对于一个长度为 $n ^{\text{∗}}$ 的排列 $p$，我们定义函数：

$$ f(p) = \sum_{i=1}^{n} \lvert p_i - i \rvert $$

给定一个数字 $n$。你需要计算当考虑从 $1$ 到 $n$ 的所有可能排列时，函数 $f(p)$ 可以取到多少个不同的值。

$ ^{\text{∗}}$ 一个长度为 $n$ 的排列是指由 $1$ 到 $n$ 的 $n$ 个不同整数按任意顺序组成的数组。例如，$[2,3,1,5,4]$ 是一个排列，但 $[1,2,2]$ 不是排列（$2$ 在数组中出现了两次），$[1,3,4]$ 也不是排列（$n=3$ 但数组中出现了 $4$）。
思路：显然$f(p)$的取值一定为偶数，所以我们只需要计算$f(p)$最大时的值就可以了(问题变为从零到最大值有多少个偶数)
显然$p$倒序排列时$f(p)$最大，$p=[n,n-1,n-2,….,3,2,1]$
当$n$为偶数时，$f(p)=2(n-1+n-3+…..+2+1)=\frac{n^{2}}{2}$
当$n$为奇数时，$f(p)=2(n-1+n-3+…..+4+2+0)=\frac{n^{2}-1}{2}$
整理一下，最后结果为$ans=\lfloor \frac{n^{2}}{4} \rfloor+1$
